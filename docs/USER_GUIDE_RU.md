## 1. Что такое система

Dream Motif Interpreter — приватная система анализа сновидений для одного пользователя. Она хранит дневник снов, помогает находить паттерны, самостоятельно образует абстрактные категории из конкретных образов и ищет параллели в мифологии и культуре.

**Принципиальная граница:** бот никогда не интерпретирует сны как психологический факт. Психологическое значение — это твоя задача. Бот даёт вычислительные наблюдения и структурированный материал для твоей собственной интерпретации.

Система приватная: бот доступен только с твоего `chat_id`.

## 2. Разговор с ботом

Пиши боту в Telegram любой вопрос о своих снах на русском или английском.

**Что бот умеет в разговоре:**

- Отвечать на вопросы об архиве снов ("Что я видел на прошлой неделе?")
- Искать по ключевым словам и смыслу ("В каких снах была вода?")
- Рассказывать о темах конкретного сна ("Какие темы в сне от 3 апреля?")
- Показывать повторяющиеся паттерны ("Какие темы встречались чаще всего?")
- Показывать абстрактные мотивы, самостоятельно выведенные из образов сна
- Искать культурные и мифологические параллели для подтверждённых мотивов

**Ограничения разговора:**

- Бот работает в рамках ограниченного tool loop (максимум 5 раундов инструментов за ответ)
- Если бот не нашёл достаточно свидетельств — честно скажет "недостаточно данных"
- Бот не додумывает: если в архиве нет ответа, он не выдумывает

## 3. Голосовые сообщения

Отправь боту голосовое сообщение — оно будет транскрибировано через OpenAI Whisper и обработано как текстовый запрос.

**Как работает:**
1. Ты отправляешь голосовое
2. Бот скачивает файл и ставит задачу транскрипции
3. Через несколько секунд возвращает ответ на то, что ты сказал

**Поддерживаемые форматы:** ogg, mp4, m4a и другие, которые принимает Telegram.

**Голосовые файлы** хранятся только во время обработки и удаляются после транскрипции.

## 4. Поиск по архиву снов

Бот использует семантический поиск (pgvector + RAG) — можно искать по смыслу, не только по точным словам.

**Примеры запросов:**
- "Покажи сны, где я куда-то падал"
- "В каких снах было ощущение тревоги?"
- "Найди сны с образами воды или моря"
- "Какие сны были самыми яркими в марте?"

**Как работает поиск:**
- Запрос переводится в embedding (OpenAI text-embedding-3-small)
- Ищется по HNSW-индексу в PostgreSQL + pgvector
- Возвращаются наиболее релевантные фрагменты с цитатами из снов
- Если ничего не найдено — бот сообщит об этом явно

## 5. Абстрактные мотивы — от образов к категории

> **Требуется:** `MOTIF_INDUCTION_ENABLED=true`

**Что это решает:** из конкретного "месива" образов сна (дорога, машина, ноги не слушаются, транспорт) система самостоятельно образует абстрактную категорию — например, *«перемещение»* или *«движение с препятствием»*. Тебе не нужно давать категорию заранее. Бот выводит её сам, абстрагируя перечисленные образы.

**Как работает индукция мотивов:**
1. `ImageryExtractor` извлекает конкретные образы из текста сна
2. `MotifInductor` формирует абстрактную мотивную метку на основе этих образов — без предустановленного словаря
3. `MotifGrounder` верифицирует, что образы действительно присутствуют в тексте сна

**Пример:** сон содержит образы *дорога, машина, не могу идти, транспорт* → система может индуцировать мотив *«движение/перемещение»* без подсказки с твоей стороны.

**Статусы мотива:** `draft` → `confirmed` / `rejected`
- Новые мотивы создаются со статусом `draft` — это вычислительное предположение, не вывод
- Ты подтверждаешь или отклоняешь мотивы через PATCH API или бота

**Через бота:**
- "Покажи мотивы сна X" — бот вызовет инструмент `get_dream_motifs`
- Мотивы со статусом `draft` представляются как предположения, не как факты
- Бот никогда не называет это "интерпретацией" — только "абстракцией" или "предположением"

**Важно:** Мотивы хранятся в `motif_inductions` — отдельно от тем `dream_themes`. Это разные слои анализа.

## 6. Культурные параллели — мифология и контекст

> **Требуется:** `RESEARCH_AUGMENTATION_ENABLED=true` и `RESEARCH_API_KEY`

**Что это решает:** для подтверждённых мотивов система ищет параллели в мифологии, фольклоре и культуре — автоматически, через внешний поиск. Например, мотив *«вылавливание рыбы → запрет её съесть»* может получить параллели: священная рыба в мифологии, табу на поедание тотемного животного, запретный плод. Бот находит это сам, не по твоему запросу на каждый случай.

**Как работает:**
1. `ResearchRetriever` выполняет внешний поиск (Tavily, max 5 результатов, timeout 5 сек)
2. `ResearchSynthesizer` (Claude) извлекает структурные параллели из найденных источников
3. Каждая параллель имеет `domain` (мифология, фольклор, литература и т.д.), `label`, `source_url`, `relevance_note` и `confidence`

**Уровни доверия (только эти три):**
- `speculative` — слабая связь, требует осторожности
- `plausible` — умеренная связь, заслуживает внимания
- `uncertain` — связь неясна

Слова `confirmed`, `verified`, `established` — запрещены. Все результаты — внешние предположения, не утверждения о твоих снах.

**Через бота:**
- Бот обязательно сообщит, что именно будет искать, и попросит подтверждение перед запросом
- После подтверждения — вызывает `research_motif_parallels`
- Результаты представляются с явным фреймингом: "внешние предположения"

**Через API:**
```
POST /motifs/{motif_id}/research   # запустить поиск
GET  /motifs/{motif_id}/research   # получить сохранённые результаты
```

**Психологическая интерпретация** — что мотив «перемещение» значит конкретно для тебя — остаётся за тобой. Система даёт структурный материал, не психологический вывод.

## 7. Оценка ответов бота

После каждого содержательного ответа бот добавляет:
> "Reply to this message to rate (1–5), or add a comment after the digit."

**Как оценить:**

Нажми **Reply** (Ответить) на сообщение бота в Telegram и напиши:
- `"4"` — оценка 4, без комментария
- `"5 Очень точно"` — оценка 5 с комментарием
- `"2 Слишком длинно"` — низкая оценка + конкретная обратная связь
- Просто текст без цифры — обрабатывается как обычный вопрос (не оценка)

Бот ответит "Thanks, noted." и сохранит оценку.

**Как твои комментарии влияют на бота:**

Бот запоминает последние комментарии и низкие оценки (1–2) и учитывает их в каждом следующем ответе. Если написал "слишком длинно" — бот постарается быть короче. Если "слишком абстрактно" — будет точнее. Без перезапуска, без дообучения — контекст встраивается в промпт перед каждым ответом.

**Fallback:** одиночная цифра без Reply тоже принимается (если бот только что ответил).

**Что не воспринимается как рейтинг:**
- "ok 3" — нет ведущей цифры
- "6" — вне диапазона
- Reply на чужое сообщение, а не на сообщение бота

Рейтинги хранятся в `assistant_feedback` и доступны через `GET /feedback` (только для чтения, требует `X-API-Key`).

## 8. Что бот не делает намеренно

| Что | Почему |
|---|---|
| Не интерпретирует сны как психологический факт | Интерпретация остаётся за тобой |
| Не изменяет архив из чата | Мутации только через REST API с явным подтверждением |
| Не использует "confirmed" для исследовательских результатов | Trust boundary: внешний поиск — это предположения |
| Не хранит мотивы как темы | Разные слои: `motif_inductions` ≠ `dream_themes` |
| Не запускает research без явного подтверждения | Confirmation-before-execution обязателен |
| Не логирует сырой текст снов | PII protection: только ID и метаданные в логах |
| Не делает неограниченный tool loop | Максимум 5 раундов инструментов |

## 9. Переменные для включения фич

По умолчанию мотивная индукция и внешний поиск **выключены**. Включение:

```env
# Мотивная индукция (Phase 9)
MOTIF_INDUCTION_ENABLED=true

# Внешний поиск параллелей (Phase 10)
RESEARCH_AUGMENTATION_ENABLED=true
RESEARCH_API_KEY=<твой_ключ_Tavily>
```

**Важно:** Изменение этих флагов требует перезапуска бота (значение кешируется при старте).

После изменения:
```bash
docker compose restart telegram-bot
```
