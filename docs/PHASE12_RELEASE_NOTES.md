# Phase 12 — Release Notes: UX fixes after first user test session

Version: 1.0
Date: 2026-04-23
Source: Тест 1 (22.04.26) — первая реальная пользовательская сессия

---

## Контекст

После завершения Phases 1–11 система прошла первую реальную тестовую сессию (Тест 1, 22.04.26).
В ходе неё были выявлены конкретные UX-проблемы: сломанные сценарии, неудобный вывод,
визуальные артефакты форматирования. Phase 12 закрывает весь этот бэклог без изменений схемы БД.

---

## Что было сломано → что исправлено

### 1. Поиск мифологических параллелей не работал вообще (WS-12.1)

**Проблема:** Инструмент `get_dream_motifs` возвращал мотивы без UUID. Когда Claude пытался
вызвать `research_motif_parallels`, у него не было `motif_id` для передачи — поиск всегда падал.

**Исправление:** Каждая строка вывода теперь содержит `[id=<uuid>]`. Claude читает его и корректно
передаёт в `research_motif_parallels`.

---

### 2. Ответы содержали markdown-разметку (`**жирный**`) (WS-12.2)

**Проблема:** Claude форматировал ответы в markdown. Telegram в plain-text режиме показывал
буквальные `**` вместо жирного текста. Выглядело как шум.

**Исправление:** В system prompt добавлен явный запрет на `**`, `*`, `__`, `[]`, `# headers`.
Для списков теперь предписан формат `1. 2. 3.`. Даты — в формате `дд.мм.гг`. Безымянный сон — `«без названия»`.

---

### 3. Список снов (`list_recent_dreams`) показывал UUID и счётчик слов (WS-12.3)

**Проблема:** Claude видел только `uuid | дата | название (N слов)`. Этого недостаточно, чтобы
сформулировать «о чём сон», и пользователь не понимал, что именно в архиве.

**Исправление:** `DreamSummary` теперь содержит `raw_text_preview` (первые 400 символов) и
`theme_names` (список тем через JOIN). UUID и `word_count` убраны из вывода инструмента.
Claude составляет краткое описание на русском в формате `«дд.мм.гг, Название: о <сюжет>»`.

---

### 4. Поиск по образу (`search_dreams`) не показывал название сна (WS-12.4)

**Проблема:** Результаты поиска содержали обрывки текста без привязки к названию сна.
Пользователь не понимал, о каком сне речь.

**Исправление:** SQL-запрос теперь пробрасывает `de.title` через все CTE (cosine, fts, fused)
до финального SELECT. `EvidenceBlock` и `SearchResultItem` получили поле `title: str | None`.
Вывод: единый нумерованный список, каждая строка содержит дату, название и вербальную
силу связи (`сильная` / `умеренная` / `слабая` по порогам 0.7 / 0.4).

---

### 5. Текст сна содержал `*` и `<` из Google Docs (WS-12.5)

**Проблема:** Сырой текст из Google Docs содержал служебные символы `*` (форматирование)
и `<` (теги), которые попадали в ответ пользователю через `get_dream`.

**Исправление:** Перед включением в вывод инструмента символы `*` и `<` удаляются.
Дополнительно лимит текста увеличен с 500 до 2000 символов — пользователь хотел читать
полный текст сна при расшифровке.

---

### 6. Бот не реагировал на естественные фразы записи сна (WS-12.6)

**Проблема:** `_is_explicit_create_request` содержал узкий список фраз. Естественные
формулировки («занеси в архив», «запиши это», «сохрани этот сон») не распознавались —
сон не сохранялся в базу.

**Исправление:** Добавлены 8 новых русских триггерных фраз:
`«сохрани этот сон»`, `«запишите»`, `«запиши это»`, `«добавь в архив»`,
`«сохрани в архив»`, `«сохранить в архив»`, `«занести в архив»`, `«занеси в архив»`.
В system prompt добавлена явная инструкция: на любую просьбу сохранить/записать/добавить
сон — всегда вызывать `create_dream`, не отвечать текстом.

---

### 7. Заголовок сна при сохранении не соответствовал ожиданиям (WS-12.7)

**Проблема:** `_resolve_dream_title` брал первую строку текста как заголовок. Результат
выглядел как обрывок записи, а не как заголовок в журнале.

**Исправление:** Новая логика формирования заголовка по четырём веткам:
- `title` + `dream_date` → `«дд.мм.гг - Название»`
- нет `title` + `dream_date` → `«дд.мм.гг, без названия»`
- `title` + нет `dream_date` → `title` как есть
- ничего → `«<сегодня дд.мм.гг>, без названия»`

---

### 8. Поток поиска параллелей был перегружен техническим преамбулом (WS-12.9)

**Проблема:** Claude перед поиском выводил длинный технический текст с подтверждением,
повторял те же мотивы, сообщал об отсутствии UUID (который теперь есть после WS-12.1),
и добавлял обобщение «Общий архетип: ...» в конце.

**Исправление:** Research Augmentation Rules в system prompt переписан:
- вместо длинного преамбула — простое предложение `«По каким мотивам провести поиск?»`
- `research_motif_parallels` вызывается только после выбора мотива пользователем
- запрещены слова `«архетип»` и `«паттерн»` (если пользователь сам не употребил)
- запрещены оценочные прилагательные (`«глубокий»`, `«интересный»`, `«мощный»`)
- запрещено добавлять обобщение в конце списка параллелей
- при пустом результате поиска — показать мотивы из `get_dream_motifs` без остановки

---

### 9. Невозможно сменить источник архива без доступа к серверу (WS-12.8)

**Проблема:** `GOOGLE_DOC_ID` задавался только через env-переменную, что требовало
перезапуска процесса и доступа к серверу.

**Исправление:** Добавлен инструмент `manage_archive_source` (`action: get | set`).
`action=get` возвращает текущий `GOOGLE_DOC_ID`. `action=set` обновляет runtime-override
без перезапуска, новое значение вступает в силу для следующего `trigger_sync`.
